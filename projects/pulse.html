PULSE

Client-site uptime monitor

You leave it running at the customer's office. It polls a list of endpoints, tracks HTTP status, response time, and TLS certificate expiry, streams live status over SSE, and draws response-time sparklines on a canvas. History is appended to a JSONL file, so there is no database to babysit.

WHAT IT DOES

ProbeHTTP / HTTPS
Metricsstatus · ms · cert
StreamSSE live
HistoryJSONL, no DB
Demo3 endpoints

STACK

Node Express SSE canvas

RUN IT

# no API key, no cloud
npm install
npm start
# watch the dashboard, then:
curl localhost:3002/api/status
REPOSITORY → zero keysruns offline

FIELD ORIGIN

After you deploy something at a client site, someone needs to know when it stops answering. Pulse is a small monitor you can leave behind: open the page, see green or red, spot a cert about to expire before it does.
