EDGESYNC

On-site data reconciliation engine

Walk into a customer site. Their customer data lives in three places that do not agree: a CSV export, a REST JSON endpoint, and a legacy fixed-width dump. EdgeSync ingests all three, reconciles them by key, and surfaces every field-level conflict with proof. It emits a clean unified dataset and a downloadable CSV through a live console.

WHAT IT DOES

SourcesCSV · REST JSON · legacy
Match keyrecord ID
Conflict viewfield-level, per source
Outputunified JSON + CSV
Demo data12 records, 4 conflicts

STACK

Python 3.11 FastAPI Uvicorn stdlib csv/json

RUN IT

# no API key, no cloud
uv venv && uv pip install fastapi uvicorn
uv run uvicorn app:app --port 8000
# open the console, then:
curl localhost:8000/api/stats
curl localhost:8000/api/export.csv
REPOSITORY → zero keysruns offline

FIELD ORIGIN

Built from the automotive recovery engagement: 1,830 records reconciled across 12 dealerships. The demo dataset is sanitized, but the reconciliation logic is the same one that matched records by ID with a 100% rate.

/ WHY IT EXISTS

The last mile of a data migration

Most "data problems" at a client site are really three systems that disagree and nobody owns the merge. EdgeSync is the tool you run after the export lands and before anyone trusts the number. It does not invent data. It shows you exactly where the sources diverge and lets you decide, with proof, which value wins.
